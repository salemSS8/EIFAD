# تقرير تحديثات الـ API لمطور الواجهات الأمامية (Frontend Report)

مرحباً بك! يوضح هذا التقرير جميع التحديثات والإضافات التي تمت مؤخراً على خادم الـ API بناءً على طلباتك والمشكلات التي أبلغت عنها. جميع هذه التحديثات متاحة حالياً على الخادم المباشر وموثقة عبر واجهة Swagger في المسار (`/api/documentation`).

---

## 1. قسم السيرة الذاتية (CV) وتعديل القيم
**المشكلة:** القدرة على تعديل القيم القديمة في السيرة الذاتية (خبرة، تعليم، لغات، مهارات...).
**الحل وطريقة الاستخدام:**
لا تحتاج لاستخدام مسارات جديدة، فالمشروع يدعم التعديل بشكل مسبق عن طريق استخدام مسارات `PUT`.
*   لتعديل بيانات خبرة محددة: `PUT /api/cvs/{cvId}/experience/{experienceId}`
*   لتعديل بيانات تعليم: `PUT /api/cvs/{cvId}/education/{educationId}`
*   لتعديل مهارة: `PUT /api/cvs/{cvId}/skills/{skillId}`
*   لتعديل لغة: `PUT /api/cvs/{cvId}/languages/{languageId}`

## 2. مشكلة الفئات وإضافة (مهارات أو لغات) جديدة غير موجودة مسبقاً
**المشكلة:** قد يُدخل المستخدم "قيمة غبية" في الفئة، أو يملك لغة/مهارة غير موجودة في الـ Dropdown.
**الحل وطريقة الاستخدام:**
تم تزويد الـ API بنقاط نهاية (Endpoints) متخصصة تتيح لك إرسال المهارات واللغات الجديدة التي يدخلها المستخدم، بحيث يرسلها الباك-إند ويحفظها، ثم يعيد لك الـ `ID` لتقوم بربطه بالسيرة الذاتية مباشرة:
*   لإضافة مهارة جديدة: `POST /api/skills` (يحتاج إلى `skill_name` و `category_id`).
   *ملاحظة الذكاء الاصطناعي: لتجنب إدخال فئات خاطئة يمكنك جلب الفئات المتاحة عبر `GET /api/skill-categories` وإجبار المستخدم على اختيار فئة، ومن ثم إضافة المهارة الجديدة بداخلها.*
*   لإضافة لغة جديدة للبرنامج: `POST /api/languages` (يحتاج إلى `LanguageName`).

## 3. إظهار "عدد المتقدمين للوظيفة"
**المشكلة:** الحاجة لإظهار كم شخص تقدم للوظيفة (للباحث وللشركة).
**الحل وطريقة الاستخدام:**
تم تضمين عدد المتقدمين بشكل تلقائي في مسارات جلب الوظائف.
عند استدعاء استعلام تفاصيل الوظيفة أو قائمة الوظائف:
*   `GET /api/jobs`
*   `GET /api/jobs/{id}`
ستجد حقلاً جديداً في كل Response Object يدعى `applications_count` يحتوي على رقم المتقدمين لهذه الوظيفة.

## 4. الفوترة والمتابعة: الشركات المتابَعة (Following Companies)
**المشكلة:** الباحث عن عمل لا يملك مساراً لاستعراض الشركات التي قام بمتابعتها.
**الحل وطريقة الاستخدام:**
تم بناء مسار جديد كلياً يقوم المطور باستدعائه لعرض قائمة الشركات المحفوظة للباحث:
*   `GET /api/companies/following` (يتطلب Bearer Auth)
يرجع مصفوفة من الشركات `CompanyProfile` مع بياناتها.

## 5. ميزات الذكاء الاصطناعي والمطابقة (AiMatch) وتوجهات السوق
**المشكلة:** المطور بحاجة لنسبة المطابقة وسببها (نقاط قوة، فجوات) بشكل مسبق، ومعرفة توجهات السوق.
**الحل وطريقة الاستخدام:**
1.  **تفاصيل المطابقة:** المسار السابق `GET /api/jobs/{jobId}/match-score` تم تحديثه، وأصبح اليوم يعيد الـ Response التالي:
    ```json
    {
      "data": {
        "job_id": 1,
        "cv_id": 3,
        "match_score": 85,
        "strengths": ["Your CV contains matching skills: PHP, Laravel", "Experience titles closely match the job role."],
        "gaps": ["No formal education listed."],
        "explanation": "Calculated heuristically based on your CV details..."
      }
    }
    ```
2.  **توجهات السوق (Market Trends):** لمعرفة أكثر المهارات طلباً في السوق وعرضها في شاشة للباحث، يمكنك استدعاء:
    *   `GET /api/market-trends`
    يرجع مصفوفة بأعلى المهارات من حيث الطلب مع تاريخ الإحصائية (`DemandCount`).

## 6. تعديل الإيميل في البروفايل
**المشكلة:** الحاجة لتمرير إيميل المستخدم وتحديثه من صفحة Profile.
**الحل وطريقة الاستخدام:**
*   في الريكويست `POST /api/profile` أو `PUT /api/profile` يمكنك الآن إرسال المتغير `Email` بجانب بيانات البروفايل الأخرى، وسيتكفل الخادم بتحديث بريد المستخدم والتحقق من أن البريد غير مستخدم من قِبل شخص آخر.

## 7. تخصيص الإشعارات (Notification Settings)
**المشكلة:** كيف يتحكم المستخدم في الإشعارات التي تصله؟
**الحل وطريقة الاستخدام:**
تم إنشاء مسارين جديدين للتحكم بحالة إشعارات المستخدم:
*   `GET /api/notifications/settings` : لجلب حالات مفاتيح (Toggles) الإشعارات (EmailNotifications, PushNotifications, JobAlerts, ApplicationUpdates, MarketingEmails).
*   `PUT /api/notifications/settings` : يرسل لتحديث الحالة، بوضع الـ field كـ True أو False.

## 8. ترتيب الوظائف حسب الشعبية المتقدمة (Most Popular)
**المشكلة:** إمكانية فلترة الوظائف لتظهر بناءً على التفاعل أو عدد التقديمات عليها.
**الحل وطريقة الاستخدام:**
*   في مسار البحث عن الوظائف `GET /api/jobs` ، يمكنك تمرير الـ query parameter المعرف بـ `sort`.
*   لتمرير الأكثر شعبية أرسل: `GET /api/jobs?sort=popular`
سيقوم الباك اند بترتيب الوظائف بناءً على الـ `applications_count` الأعلى.

---
> [!NOTE]
> لا تتردد في مراجعة ملف **Swagger Collection** الموجود في الخادم من مسار `/api/documentation` لاختبار هذه الميزات والحصول على وصف مفصل للـ Payload والـ Headers المرفوعة.
